# 2026-05-14 iOS Webhook Streaming Refactor

## Summary

Refactored the iOS webhook path to match the Mac app's append-only
typing semantics.

New behavior:

- Stable transcript text can stream to the webhook during speech as
  append-only deltas.
- End-of-utterance sends a final event that appends only any missing
  remainder plus a submit newline.
- Deepgram diarized speaker attribution is a suffix, not a prefix.
- The suffix style mirrors the Mac final-submit case:

```text
\n[Speaker]\n
```

When no diarized speaker is available, the final webhook append is just
the submit newline.

## Protocol

`DiarizedUtterance` now includes `event_type` and `append_text`.

Live stream event:

```json
{
  "event_type": "stream_delta",
  "append_text": "new words",
  "is_final": false
}
```

EOU final event:

```json
{
  "event_type": "utterance_end",
  "append_text": "\n[Alex]\n",
  "is_final": true
}
```

If stable text did not already stream before EOU, the final
`append_text` includes the missing text before the suffix.

## Relay

The `yapatron-relay` endpoint now prefers `append_text` and pastes it
directly, including LF newlines that are intentionally generated by the
app. Legacy final-only payloads without `append_text` still format as
`[Speaker] text\n`.

The relay rate limit was increased to handle short streaming deltas
instead of only finalized utterances.

## Follow-Up

Opened GitHub issue #5 after confirming the current iOS Deepgram path
streams microphone buffers continuously while Listening, including
silent periods. The Mac app has local speech gating before downstream
send; iOS needs that later, but it was kept out of this patch.
